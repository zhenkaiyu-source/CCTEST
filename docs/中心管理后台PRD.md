## 2. 项目背景与目标

### 2.1 项目背景

瓷韵平台目前已完成创造者端的开发，支持艺术家上传作品、管理订单、查看财务数据等功能。随着平台规模扩大，需要建设**中心管理后台**以实现对平台整体的运营管理，包括艺术家管理、作品审核、订单监控、数据分析等功能。

### 2.2 产品目标

- **核心目标**：建立完整的平台运营管理能力
- **用户群体**：平台管理员、运营人员
- **核心价值**：
  - 统一管理平台所有艺术家和作品
  - 实时监控平台运营数据
  - 确保平台内容合规
  - 提升运营效率

#### 5.1.3 认证审核

| 功能描述 | 审核艺术家提交的认证申请 |
|----------|--------------------------|
| 访问路径 | `/admin/artists/verify` |
| 请求方式 | PUT `/api/admin/artists/:id/verify` |

**待审核列表字段**：

| 字段 | 说明 |
|------|------|
| 申请时间 | 格式: YYYY-MM-DD HH:mm |
| 艺术家 | 头像 + 昵称 + 手机号 |
| 认证类型 | 个人认证 / 企业认证 |
| 认证材料 | 身份证明、手持证件照、技艺证明等 |
| 申请说明 | 艺术家填写的认证说明 |
| 操作 | 通过 / 拒绝（需填写原因） |

**审核流程**：
1. 管理员查看认证材料
2. 选择「通过」或「拒绝」
3. 如拒绝，需填写拒绝原因（会通知艺术家）
4. 提交后系统发送通知给艺术家
5. 记录审核日志

|----------|------------------------------|

**审核项目**：

| 项目 | 检查点 |
|------|--------|
| 图片合规 | 无水印、无敏感内容、无盗图 |
| 描述合规 | 无违禁词、无虚假宣传 |

- 审核通过 → 状态变为「已上架」
- 审核拒绝 → 填写原因，状态变为「已下架」
- 标记待整改 → 退回给艺术家修改

| 艺术家已提现 | 历史已提现总额 | SUM(已完成提现) |

## 6. API 接口设计

### 6.1 新增 Admin API

```
# 艺术家管理
GET    /api/admin/artists              # 艺术家列表
GET    /api/admin/artists/:id          # 艺术家详情
PUT    /api/admin/artists/:id          # 编辑艺术家
PUT    /api/admin/artists/:id/status   # 修改状态
PUT    /api/admin/artists/:id/verify   # 认证审核

# 作品管理
GET    /api/admin/works                # 作品列表
GET    /api/admin/works/:id            # 作品详情
PUT    /api/admin/works/:id            # 编辑作品
PUT    /api/admin/works/:id/status     # 修改状态
DELETE /api/admin/works/:id            # 删除作品

# 订单管理
GET    /api/admin/orders               # 订单列表
GET    /api/admin/orders/:id           # 订单详情
PUT    /api/admin/orders/:id/status    # 修改状态

# 财务管理
GET    /api/admin/finance/overview     # 财务概览
GET    /api/admin/finance/transactions # 交易流水
GET    /api/admin/finance/withdrawals  # 提现列表
PUT    /api/admin/finance/withdrawals/:id # 提现审核

# 数据分析
GET    /api/admin/analytics/dashboard # 数据看板
GET    /api/admin/analytics/ranking    # 排行榜

# 系统设置
GET    /api/admin/settings/basic       # 基础设置
PUT    /api/admin/settings/basic       # 更新设置
GET    /api/admin/settings/categories  # 分类列表
```

### 6.2 认证中间件

```javascript
// Admin 权限校验中间件
const adminAuth = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (!token) return res.status(401).json({ error: '未登录' });

  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    // 校验用户角色
    if (!['super_admin', 'ops_admin', 'finance_admin', 'support'].includes(decoded.role)) {
      return res.status(403).json({ error: '权限不足' });
    }
    req.userId = decoded.userId;
    req.userRole = decoded.role;
    next();
  } catch (err) {
    return res.status(401).json({ error: '登录已过期' });
  }
};
```

## 11. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| MongoDB 未安装 | 数据无法持久化 | 继续使用内存数据库，或用户提供 MongoDB 连接串 |
| 现有代码冲突 | 重复开发 | 复用现有组件和样式，保持一致性 |
| 大数据量性能 | 列表加载慢 | 优化分页、增加索引 |
| 安全漏洞 | 数据泄露 | 接口权限校验、操作日志记录 |
